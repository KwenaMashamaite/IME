pure virtual

Update state in fixed time steps.

Parameters
deltaTimeTime passed since last update

This function will be called multiple times per frame by the engine. The delta passed to it is always the same. All update that require a fixed timestep must be defined in this function. This function allows updates to be the same regardless of how fast or slow the computer is

virtual bool IME::State::isInitialized ( ) const
pure virtual

Check if a state is initialized or not.

Returns
True if state is initialized or false if state is not initialized

This function will be called by the engine when a state push or a state pop was requested in the previous frame. This ensures that a previously initialized state is resumed instead of being reinitialized and vice versa

See also
initialize(), resume() and pause()

◆ pause()

virtual void IME::State::pause ( )
pure virtual

Pause the state.

This function will be called by the game engine on the current state when a state change is requested. This function allows a state to pause itself such that when it is returned to, it can resume where it left of instead of being reinitialized

virtual void IME::State::update ( float  deltaTime)
pure virtual

Update the state.

Parameters
deltaTimeTime passed since last update

This function will be called once per frame by the engine. The delta passed to it is frame rate dependent. This means that it depends on how long the current frames takes to complete. All updates that need a variable timestep must be defined in this function
