\doxysubsection{Detailed Description}
A class for publishing system events. 

These events are \+:


\begin{DoxyEnumerate}
\item closed -\/ Fired when a request to close the application window is made. Passes no arguments on notify
\item key\+Pressed -\/ Fired when a key is pressed. Passes the key (\mbox{\hyperlink{class_keyboard_a79a4b804a620cfbf3acd672dae2fb7a1}{Keyboard\+::\+Key}}) that was pressed as an argument on notify
\item key\+Released -\/ Fired when a key is released. Passes the key (\mbox{\hyperlink{class_keyboard_a79a4b804a620cfbf3acd672dae2fb7a1}{Keyboard\+::\+Key}}) that was released as an argument on notify
\item mouse\+Moved -\/ Fired when the mouse is moved. Passes the x and y coordinate of the mouse cursor as arguments on notify (int, int)
\item mouse\+Button\+Pressed -\/ Fired when a mouse button is pressed. Passes the button (\mbox{\hyperlink{class_mouse_a2189c95ac90001266b97105c25085270}{Mouse\+::\+Button}}) that was pressed as an argument on notify
\item mouse\+Button\+Released -\/ Fired when a mouse button is released. Passes the button (\mbox{\hyperlink{class_mouse_a2189c95ac90001266b97105c25085270}{Mouse\+::\+Button}}) that was released as an argument on notify
\end{DoxyEnumerate}

\begin{DoxyNote}{Note}
When adding listeners to an event, the the parameters of the callback function must match exactly with the arguments an event provides on notify, otherwise the function will never be called when the event is raised. In addition, if an event listener is added to an event that\textquotesingle{}s none of the above events, that event will never be raised, and consequently the callback will never be invoked 
\end{DoxyNote}

Process system events. 

This function will notify event listeners if any events have been captured by the system. Note, an invocation of this function wil empty the system event queue. All instances of the class share the same event queue. This means that, if one instance calls this function, then the event queue will be processed for all instances. However, only event listeners registered with the instance that invoked the function will be notified
